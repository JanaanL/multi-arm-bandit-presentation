Pure Exploitation vs Pure Exploration

The two extremes — one commits too early, one never commits at all.
Pure Exploitation cycles through machines A → B → C until the first reward lands, then commits to that machine forever. It's the most aggressive possible commitment strategy — one lucky pull locks in the choice, regardless of whether a better machine exists.

Pure Exploration picks randomly every single pull. It learns a lot about all three machines over time, but wastes its budget on machines it already knows are worse.

Stage 2

Epsilon-Greedy

A simple but powerful balance: mostly exploit, occasionally explore.
Epsilon-Greedy picks the currently best-looking machine (1 − ε) of the time, and picks randomly ε of the time. It keeps exploring indefinitely — which means it can self-correct if it made a bad early decision. Watch the pull bars: it mostly converges on one machine but keeps small amounts of exploration running throughout.

Stage 3

Upper Confidence Bound (UCB)

Optimism in the face of uncertainty — no epsilon to tune.
UCB scores each machine as observed win rate + confidence bonus. The bonus is large when a machine has been pulled few times and shrinks as data accumulates. Machines you know little about get explored automatically — and once you're confident a machine is bad, you stop wasting pulls on it. No epsilon to set.

Watch the pull bars carefully: UCB starts by spreading across all three machines, then gradually concentrates on the best one as confidence grows.

Stage 4

Thompson Sampling

Bayesian learning — watch the algorithm's beliefs update in real time.
Thompson Sampling maintains a probability distribution over each machine's true payout rate — wide at first (high uncertainty), narrowing with every observation. Each pull it samples a random value from each distribution and picks the highest. Early on the wide distributions cause natural exploration. As data accumulates the distributions tighten and the algorithm hones in on the best machine.
